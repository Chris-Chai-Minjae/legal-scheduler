## 1. 개요

### 목표
법률 블로그 글의 네이버 검색 노출을 극대화하기 위한 **SEO 자동 평가 점수화 + 즉시 적용 시스템**을 구축한다. 사용자가 블로그 글을 작성하거나 편집할 때 SEO 점수를 실시간으로 확인하고, AI의 제안을 한 번의 클릭으로 수용할 수 있는 효율적인 워크플로우를 제공한다.

### 문제 정의
- 법률 실무자들이 작성한 블로그 글이 네이버 검색에 노출되지 않음
- SEO 최적화 방법을 모르거나 시간이 없음
- 기존 글을 수동으로 최적화하는 것은 번거로움

### 해결책
- 글 상세 페이지에 SEO 평가 패널 자동 표시
- 8개 항목별 점수화 및 시각화 (100점 만점)
- AI 기반 항목별 재작성 제안
- 한 번의 클릭으로 제안 사항 적용

## 2. 핵심 기능

### 2.1 SEO 자동 분석
- **트리거**: 블로그 글 상세(show) 페이지 진입 시 자동 실행
- **분석 대상**: 제목, 메타 디스크립션, 본문, H2 소제목, URL 슬러그, 이미지 alt 태그, 내부 링크, 메타 태그, 글 길이, 가독성
- **결과 저장**: 분석 결과를 BlogPost 모델의 seo_score(integer), seo_details(jsonb)에 저장

### 2.2 점수화 대시보드
- **위치**: 글 상세 페이지 오른쪽 사이드 패널
- **구성**:
  - 총점 게이지 (원형 프로그레스, 0~100)
  - 카테고리별 섹션 (키워드 최적화 40점, 기술적 SEO 40점, 보너스 20점)
  - 각 항목별 점수 및 현재 상태 표시
  - "적용", "전체 최적화" 액션 버튼

### 2.3 항목별 AI 재작성
- **인터페이스**: 각 항목의 "적용" 버튼 클릭 → 모달/인라인 비교 화면
- **비교 화면**: 현재 값 vs 제안된 값
- **확인 후 교체**: "확인" 버튼 클릭 시 본문 즉시 업데이트
- **Turbo Stream**: 실시간 UI 동기화 (새로고침 불필요)

### 2.4 전체 최적화
- **버튼**: "모든 항목 최적화" 또는 "일괄 최적화"
- **동작**: 모든 8개 항목을 순차적으로 분석 및 제안
- **적용**: 사용자 확인 후 한 번에 적용

## 3. 사용자 스토리

### Story 1: 신규 글 작성 후 SEO 점수 확인
```gherkin
Given 법률 실무자가 AI로 글을 생성했을 때
When 생성된 글 상세 페이지에 진입하면
Then SEO 평가 패널이 자동으로 나타나고 점수(예: 65/100)를 보여준다
And 부족한 부분(예: 메타 디스크립션 미작성)이 빨간색으로 표시된다
```

### Story 2: 특정 항목 개선
```gherkin
Given SEO 평가 패널에서 "제목 키워드" 항목이 8/10점일 때
When "적용" 버튼을 클릭하면
Then 모달이 열려 현재 제목과 제안된 제목을 비교할 수 있다
And "확인" 버튼을 클릭하면 제목이 변경되고 점수가 올라간다
```

### Story 3: 전체 최적화
```gherkin
Given 기존 글의 SEO 점수가 50/100일 때
When "모든 항목 최적화" 버튼을 클릭하면
Then 8개 항목 모두에 대한 제안이 순차적으로 표시된다
And 사용자는 각 제안을 개별 확인하거나 모두 수용할 수 있다
And 완료 후 총점이 85/100으로 올라간다
```

### Story 4: 초안 저장 및 복구
```gherkin
Given 글 편집 중 SEO 최적화를 시작했을 때
When 페이지를 벗어나면
Then "변경 사항을 저장하시겠습니까?" 알림이 표시된다
And 저장 시 최적화된 내용이 데이터베이스에 반영된다
```

## 4. SEO 평가 항목 (네이버 최적화 기준)

### 4.1 키워드 최적화 (40점)

#### 제목 키워드 (10점)
- **기준**:
  - 32자 이내
  - 핵심 키워드가 제목 앞부분(첫 3단어)에 배치
  - 특수문자 최소화
- **예시**:
  - ✓ "기업 소송 절차: 변론일 일정 관리법"
  - ✗ "변론일이란? 기업 소송 중 가장 중요한 것"

#### 메타 디스크립션 (10점)
- **기준**:
  - 80자 이내 (네이버 스니펫 최적화)
  - 제목의 핵심 키워드 포함
  - 행동 유도(CTA) 포함
- **예시**:
  - ✓ "기업 소송의 변론일 일정을 효율적으로 관리하는 방법을 알아보세요."
  - ✗ "이 글은 기업 소송에 대한 글입니다."

#### 키워드 밀도 (10점)
- **기준**:
  - 타겟 키워드가 본문 내 1.5~2.5% 비율로 나타남
  - 과다 반복 금지 (스팸 판정)
- **예시**: 1000자 글에서 타겟 키워드 15~25회 등장

#### H2 소제목 구조 (10점)
- **기준**:
  - 3개 이상의 H2 소제목
  - 각 H2에 타겟 키워드 또는 관련 키워드 포함
  - 계층 구조 명확 (H2 → H3 → 본문)
- **예시**:
  - ✓ "## 변론일 일정 확인하는 방법"
  - ✗ "## 소개"

### 4.2 기술적 SEO (40점)

#### URL 구조/슬러그 (10점)
- **기준**:
  - 한글 슬러그 또는 영문 키워드 슬러그 (하이픈 구분)
  - 과도하게 길지 않음 (60자 이내)
  - 날짜나 버전 번호 미포함
- **예시**:
  - ✓ `/blog/변론일-일정-관리법` 또는 `/blog/trial-date-management`
  - ✗ `/blog/post-12345` 또는 `/blog/trial-2026-04-06-news`

#### 이미지 Alt 태그 (10점)
- **기준**:
  - 최소 1개 이상의 이미지
  - 모든 이미지에 설명적인 alt 태그 작성
  - alt 태그에 키워드 자연스럽게 포함
- **예시**:
  - ✓ `alt="기업 소송 변론일 달력 일정 예시"`
  - ✗ `alt="image" 또는 alt="pic123"`

#### 내부 링크 (10점)
- **기준**:
  - 같은 주제 글 2개 이상 연결
  - 앵커 텍스트가 설명적임
  - 링크 대상이 관련성 높음
- **예시**:
  - ✓ "변론일 준비 서면은 [이 글](...)을 참고하세요."
  - ✗ "자세히 보기"

#### 메타 태그 최적화 (10점)
- **기준**:
  - OG 태그 작성 (og:title, og:description, og:image)
  - canonical 태그 명시
  - 모바일 viewport 설정
- **예시**:
  ```html
  <meta property="og:title" content="기업 소송 변론일 일정 관리법">
  <meta property="og:description" content="변론일 일정을 효율적으로 관리하는 법...">
  <canonical href="https://example.com/blog/변론일-일정-관리법">
  ```

### 4.3 보너스 (20점)

#### 글 길이 (10점)
- **기준**:
  - 2,000자 이상 (10점)
  - 1,500~2,000자 (7점)
  - 1,000~1,500자 (4점)
  - 1,000자 미만 (1점)

#### 가독성 (10점)
- **기준**:
  - 평균 문장 길이 40자 이내
  - 문단 길이 3~5개 문장
  - 단락 사이 여백 있음
  - 불릿 포인트/숫자 리스트 활용
- **예시**:
  - ✓ "변론일을 관리하는 3가지 방법이 있습니다. 첫째, ..."
  - ✗ "변론일을 관리하는 것은 매우 중요하고 복잡하며 많은 시간이 필요하고 전문 지식이 필요합니다."

## 7. 제약 조건

1. **기존 기능 유지**: 블로그 show 페이지 외의 index, new, edit 페이지는 변경 없음
2. **레이아웃**: 70% 본문 + 30% SEO 패널로 제한 (반응형 모바일에서는 하단 탭)
3. **API 추가만**: FastAPI blog-ai 백엔드에만 엔드포인트 추가 (새로운 마이크로서비스 금지)
4. **도메인 제외**: B(콘텐츠 품질), D(법률 도메인)는 AI가 판단하지 않음 (사용자 책임)
5. **응답 시간**: SEO 분석 5초 이내, 재작성 제안 10초 이내

## 8. 범위 (In/Out)

### In Scope
- SEO 분석 엔드포인트 개발 (FastAPI)
- SEO 재작성 제안 엔드포인트 (FastAPI)
- BlogPost 모델 컬럼 추가 (seo_score, seo_details)
- SEO 패널 UI 컴포넌트 (Rails Hotwire)
- Stimulus Controller (seo_panel_controller.js)
- 비교 모달 또는 인라인 프리뷰
- Turbo Stream 실시간 업데이트

### Out of Scope
- 네이버 검색 API 직접 연동
- Google Analytics 자동 연동
- 백링크 분석
- 경쟁사 키워드 분석
- 콘텐츠 표절 검사
- 법률 도메인 전문성 검증
