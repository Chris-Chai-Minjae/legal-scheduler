## 2. 주요 사용자 시나리오

### Scenario 1: 신규 글 작성 후 SEO 점수 확인

**사용자**: 박준호 (43세 변호사, AI 블로그 기능 애용)

**컨텍스트**:
- 기존 AI 블로그 기능으로 "기업 소송 변론일 관리법" 글 생성 완료
- 글 상세 페이지 진입 (자동으로 show 페이지로 이동)

**흐름**:

```
1. [진입] show 페이지 로드
   └─ Rails: 글 데이터 렌더링
   └─ Hotwire: _seo_panel.html.erb 포함

2. [로드] SEO 패널 Stimulus 컨트롤러 초기화
   └─ "분석 중..." 상태 표시

3. [분석] 자동으로 FastAPI 호출
   POST /api/seo/analyze
   ├─ content: "기업 소송의 변론일은..."
   ├─ title: "기업 소송 변론일 일정 관리법과 실무 팁"
   ├─ description: null (미작성)
   ├─ slug: "변론일-일정-관리법"
   └─ target_keywords: ["변론일", "기업 소송"]

4. [응답] FastAPI 분석 완료
   └─ { score: 65, items: [...] }

5. [저장] Rails가 BlogPost에 저장
   └─ seo_score: 65
   └─ seo_details: { ... }

6. [렌더링] Turbo Stream으로 패널 업데이트
   └─ SEO 점수 게이지 표시 (원형 게이지, 65/100)
   └─ 8개 항목 모두 표시
   └─ 빨간색 (0~4점), 노란색 (5~7점), 초록색 (8~10점)

7. [화면] 사용자 확인
   ├─ 총점 65/100
   ├─ 키워드 최적화: 25/40
   │  ├─ 제목 키워드: 8/10 (주황색 - 32자 제한 초과)
   │  ├─ 메타 디스크립션: 0/10 (빨간색 - 미작성)
   │  ├─ 키워드 밀도: 7/10 (주황색 - 1.2% < 1.5%)
   │  └─ H2 소제목: 10/10 (초록색)
   ├─ 기술적 SEO: 24/40
   │  ├─ URL 슬러그: 10/10
   │  ├─ 이미지 Alt: 5/10
   │  ├─ 내부 링크: 3/10
   │  └─ 메타 태그: 6/10
   └─ 보너스: 18/20
      ├─ 글 길이: 10/10
      └─ 가독성: 8/10
```

**사용자의 선택**:
- "메타 디스크립션" [적용] 버튼 클릭 (가장 큰 개선 기대)

### Scenario 2: 메타 디스크립션 AI 제안 받아들이기

**사용자**: 박준호

**컨텍스트**:
- SEO 패널에서 메타 디스크립션 항목의 [적용] 버튼 클릭

**흐름**:

```
1. [클릭] "메타 디스크립션" [적용] 버튼
   └─ Stimulus 이벤트 발생
   └─ optimizeItem('meta_description')

2. [요청] FastAPI에 재작성 제안 요청
   POST /api/seo/optimize/meta_description
   ├─ content: "기업 소송의 변론일은..."
   ├─ title: "기업 소송 변론일 일정 관리법과 실무 팁"
   ├─ description: null
   └─ target_keywords: ["변론일", "기업 소송"]

3. [AI 처리] Claude가 메타 디스크립션 생성
   "변론일 일정을 효율적으로 관리하는 방법을 알아보세요."

4. [응답] FastAPI 반환
   {
     "item_id": "meta_description",
     "current_value": null,
     "suggested_value": "변론일 일정을 효율적으로 관리하는 방법을 알아보세요.",
     "ai_reasoning": "메타 디스크립션이 없어서 추가했습니다. 80자 이내..."
   }

5. [모달] 비교 화면 표시
   ┌─────────────────────────────────┐
   │ 메타 디스크립션 비교             │
   ├─────────────────────────────────┤
   │ 현재 값: (없음)                  │
   │                                 │
   │ 제안된 값:                       │
   │ "변론일 일정을 효율적으로        │
   │  관리하는 방법을 알아보세요."    │
   │ (34자)                          │
   │                                 │
   │ AI 설명:                        │
   │ 메타 디스크립션이 없어서 추가... │
   │                                 │
   │ [취소] [확인]                    │
   └─────────────────────────────────┘

6. [확인] 사용자 "확인" 버튼 클릭
   └─ PATCH /blog_posts/:id/seo/apply
   ├─ item_id: "meta_description"
   ├─ field_name: "description"
   └─ new_value: "변론일 일정을 효율적으로 관리하는 방법을 알아보세요."

7. [업데이트] Rails가 BlogPost 업데이트
   └─ description: "변론일 일정을..."

8. [재분석] SEO 점수 재계산
   POST /api/seo/analyze (업데이트된 내용)
   └─ 새로운 점수: 75/100

9. [저장] BlogPost 저장
   └─ seo_score: 75
   └─ seo_details: { ... }

10. [렌더링] Turbo Stream으로 즉시 업데이트
    └─ 본문에 메타 디스크립션 표시 (head 태그에 반영)
    └─ SEO 패널 점수 업데이트 (65 → 75)
    └─ 메타 디스크립션 항목 10/10으로 변경 (초록색)

11. [모달 닫기] 비교 모달 자동 닫기
    └─ 사용자는 즉시 업데이트된 글 + 상승한 SEO 점수 확인
```

**사용자의 다음 선택**:
- 다른 항목들도 최적화할지 판단
- "모든 항목 최적화" 버튼을 클릭할 수도 있음

### Scenario 3: 전체 최적화 (일괄 처리)

**사용자**: 박준호

**컨텍스트**:
- 현재 글의 SEO 점수 65/100
- 시간이 없어서 빠르게 모든 항목을 최적화하고 싶음

**흐름**:

```
1. [클릭] "모든 항목 최적화" 버튼
   └─ Stimulus 이벤트 발생
   └─ optimizeAllItems()

2. [확인 모달] "이 작업은 여러 항목을 순차적으로 최적화합니다.
   시간이 약 30초 소요됩니다. 계속하시겠습니까?"
   │
   ├─ [취소]: 작업 중단
   └─ [계속]: 다음 단계로

3. [로드] 진행도 표시
   ┌─────────────────────────────┐
   │ 최적화 진행 중 (1/8)        │
   │ ========================== │ (1/8)
   │ 현재 처리: 제목 키워드...   │
   └─────────────────────────────┘

4. [순차 처리] 각 항목별로 FastAPI 호출
   First iteration:
   ├─ POST /api/seo/optimize/title_keyword
   ├─ [비교 모달] 표시 (또는 자동 적용 옵션)
   ├─ 사용자 선택: "적용", "건너뛰기", "나중에"
   └─ 결과 저장

   Second iteration:
   ├─ POST /api/seo/optimize/meta_description
   ├─ ...
   └─ ...

   (8번 반복)

5. [완료] 모든 항목 처리 완료
   ┌─────────────────────────────┐
   │ 최적화 완료!                │
   │ ========================== │ (8/8)
   │ 최종 점수: 85/100           │
   │ 개선도: +20점               │
   │                             │
   │ [닫기] [세부사항 보기]       │
   └─────────────────────────────┘

6. [렌더링] 최종 결과 표시
   └─ 모든 항목 업데이트
   └─ 총점 상승 (65 → 85)
   └─ 적용된 항목 표시
```

**사용자의 경험**:
- 30초 만에 주요 SEO 항목 자동 최적화
- 총점 20점 상승
- 네이버 검색 노출 기대

### Scenario 4: 글 편집 후 수동 최적화

**사용자**: 김영미 (35세 로펌 마케팅 담당자)

**컨텍스트**:
- 기존 글을 읽으며 일부 내용을 수정하고 싶음
- 수정 후 SEO 영향도를 확인하고 싶음

**흐름**:

```
1. [편집] 본문 내용 수정
   ├─ 글 길이 증가 (1500자 → 2200자)
   ├─ 새로운 H2 소제목 추가
   └─ 인라인 편집 후 저장

2. [트리거] 저장 버튼 클릭
   └─ Rails가 BlogPost 업데이트

3. [재분석] 자동으로 SEO 분석 재실행
   └─ 백그라운드 작업으로 처리 (비동기)
   └─ "분석 중..." 상태 표시

4. [완료] 새로운 분석 결과 수신
   └─ 점수 상승 (65 → 72)
   └─ 몇 가지 항목 개선됨
   └─ 새 항목 추가됨 (예: H2 소제목 최적화 가능)

5. [패널 업데이트] Turbo Stream으로 즉시 반영
   └─ 상승한 점수 시각화
   └─ 개선된 항목 초록색으로 표시
   └─ 새 항목 추가 제안 표시

6. [사용자 선택] 추가 최적화 여부 판단
   ├─ "그만": 현재 점수 (72)로 마무리
   └─ "계속": 새로운 항목들 최적화
```

## 3. 상세 UI/UX 스펙

### 3.1 SEO 패널 (우측 30%)

**데스크톱 (1200px+)**:
- 고정 우측 사이드바
- 너비: 300px (유동적 가능)
- 스크롤: 글 내용과 독립적으로 스크롤 가능
- 배경: 흰색 또는 라이트 그레이

**태블릿 (768px ~ 1199px)**:
- 하단 탭 또는 드로어로 변경
- 클릭 시 확장, 다시 클릭 시 축소

**모바일 (< 768px)**:
- 하단 플로팅 버튼 (FAB)
- "SEO 점수: 65/100"
- 클릭 시 풀스크린 모달로 확장

### 3.2 점수 게이지

```
┌──────────────────────┐
│  SEO 점수            │
│                      │
│     ⬤ 65            │  ← 원형 프로그레스바 (SVG)
│    /   \            │     색상: 빨강(0~40), 노랑(40~70), 초록(70~100)
│   |  65 |            │     텍스트: 현재 점수 / 최대 점수
│   |/100 |            │
│    \   /            │
│     ⬜              │
│                      │
│ 개선도: ↑ 15점      │  ← 이전 점수 대비 증감 표시
│                      │
└──────────────────────┘
```

**색상 규칙**:
- 🔴 빨강 (0~40점): 긴급 (error)
- 🟠 주황 (40~70점): 경고 (warning)
- 🟢 초록 (70~100점): 좋음 (success)

## 6. 접근성 (Accessibility)

### 6.1 키보드 네비게이션
- Tab 키로 모든 버튼 순회 가능
- 엔터 또는 스페이스로 활성화
- 모달 내에서 Esc 키로 닫기

### 6.2 스크린 리더
- 모든 항목에 aria-label 추가
- 점수 변화 알림 (aria-live="polite")
- 모달 포커스 트래핑

### 6.3 색상 대비
- 모든 텍스트 WCAG AA 이상 대비
- 색상만으로 정보 전달 금지 (아이콘 추가)
