|------|------|-----------|

### 3.2 동시성 제약

```

├─ 단일 사용자: 최대 2개 분석 동시 (순차 큐 추천)

└─ FastAPI 워커: 4개 (Uvicorn)

AI 생성 동시성:
├─ Claude API: 분당 최대 10개 토큰/밀리초
├─ 각 사용자: 분당 10개 최적화 요청 제한
└─ 큐 대기: 30초 이상 경고
```

### 3.3 리소스 제약

```
메모리:
├─ FastAPI 프로세스: 512MB ~ 1GB
├─ Rails 프로세스: 256MB ~ 512MB
└─ PostgreSQL: 2GB 공유 버퍼

디스크:
├─ seo_details JSONB: 평균 3KB/글
├─ 월 10,000개 글 기준: ~30MB/월

네트워크:
├─ 평균 요청 크기: 5KB
├─ 평균 응답 크기: 10KB
```

## 4. 보안 제약

### 4.1 인증/인가

```
Rails:
├─ Devise 기존 세션 유지
├─ 로그인 사용자만 SEO 기능 접근 가능
├─ 글 소유자 또는 관리자만 수정 가능
└─ API 토큰 인증 (향후 추가)

FastAPI:
├─ 인증 없음 (내부 네트워크, Rails 프록시만)
├─ IP 화이트리스트 검토 (향후)
└─ API 키 (향후 추가 가능)
```

### 4.2 데이터 보안

```
저장 (At Rest):
├─ PostgreSQL 암호화 (pg_trgm, pgcrypto 모듈)
├─ 백업 암호화 (AES-256)
└─ PII 없음 (법률 도메인 데이터만)

전송 (In Transit):
├─ Rails ← → 브라우저: HTTPS + HSTS
├─ Rails ← → FastAPI: HTTPS 또는 내부 네트워크
├─ FastAPI ← → Claude: HTTPS (공식 SDK)
└─ CSRF 토큰 필수 (POST, PATCH, DELETE)

처리 (In Use):
├─ 민감한 데이터 메모리 암호화 금지 (성능)
├─ 로그 파일에서 PII 제외
└─ 오류 메시지에서 경로/구조 노출 금지
```

### 4.3 입력 검증 (XSS, SQL Injection 방지)

```
FastAPI:
├─ Pydantic 자동 검증
├─ 최대 길이 제한: content 100,000자, title 255자
├─ 특수문자 필터링 (선택사항, HTML 태그 제거)
└─ 레이트 리미팅 활성화

Rails:
├─ Strong Parameters 사용
├─ HTML sanitize (user input)
├─ Turbo Stream 자동 escape
└─ Content Security Policy 헤더
```

### 4.4 권한 검증

```ruby
# before_action: authorize_user!
class Blog::SeoController < ApplicationController
  before_action :set_blog_post
  before_action :authorize_user!

  def authorize_user!
    unless current_user == @blog_post.user || current_user.admin?
      render json: { error: "Unauthorized" }, status: 403
    end
  end
end
```

## 7. 비기능 요구사항 (NFR)

```
목표: 99.9% (월 45분 다운타임 허용)

└─ 계획된 유지보수: SLA 제외

├─ 심각도 1 (완전 다운): 15분 이내 통지
├─ 심각도 2 (기능 손상): 1시간 이내 통지
├─ 심각도 3 (경미): 4시간 이내 통지
└─ 복구 목표: 심각도별로 30분, 2시간, 8시간
```

### 7.2 확장성 (Scalability)

```
수평 확장:
├─ 사용자 증가: 500명 → 5,000명 (10배)
├─ 월간 글 증가: 1,000개 → 10,000개 (10배)
└─ 동시 연결: 50명 → 500명 (10배)

성능 저하 허용 범위:
├─ 응답 시간: < 20% 증가
├─ 에러율: 변화 없음
└─ CPU/메모리: 80% 이하 사용률 유지

제약 사항:
├─ 수직 확장 (더 큰 서버)는 선택사항
├─ 수평 확장 (더 많은 인스턴스)은 반드시 지원
└─ 무상태 설계 필수 (로컬 파일 저장 금지)
```

