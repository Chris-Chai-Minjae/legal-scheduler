## 1. 페르소나 개요

SEO 평가 시스템의 주 사용자는 법률 분야 전문가들입니다. 그들은 기술에 어느 정도 친숙하지만, SEO 최적화는 본업이 아닙니다. 따라서 빠르고 직관적인 인터페이스가 중요합니다.

**전체 사용자 분포**:
```
┌─────────────────────────────────┐
│ 사용자 세분화                   │
├─────────────────────────────────┤
│ Primary (주요):                 │
│ ├─ 개인 변호사 (40%)            │
│ └─ 로펌 변호사 (30%)            │
│                                 │
│ Secondary (부가):               │
│ ├─ 로펌 마케팅 담당자 (20%)     │
│ └─ 법무팀장/관리자 (10%)        │
└─────────────────────────────────┘
```

```
전체 점수: 7/10 (높음)

세부:
├─ 컴퓨터 기본: 9/10 (PC 자유자재 사용)
├─ 웹 기술: 6/10 (마크다운, HTML 기본)
├─ AI 도구: 8/10 (ChatGPT, Claude 매일 사용)
├─ 모바일 앱: 7/10 (iOS, 앱스토어 이용)
└─ 클라우드 서비스: 7/10 (구글 워크스페이스, 드롭박스)
```

### 2.4 Goal 및 Motivation

```
주요 목표:
├─ 블로그 글로 신규 고객 확보
│  └─ "기업 소송" 검색 시 노출 최상위
├─ 생각을 효율적으로 정리/공유
│  └─ 글쓰기 시간 단축 (현재 1시간/글)

   └─ "신뢰할 수 있는 정보 제공자"

숨겨진 니즈:
├─ 글의 법률적 정확성 검증 (AI 활용)
├─ 경쟁사 변호사 글과의 비교
├─ 자신의 글이 정말 영향력 있는지 확인
└─ 글 수정 후 효과 측정 (피드백)
```

### 2.5 Pain Points (문제점)

```
기술적:
├─ SEO 최적화 방법을 모름
│  └─ "어떻게 네이버 첫 페이지에 올려야 하는가?"
├─ 기존 글을 수정하는 것이 번거로움
├─ 글의 품질을 객관적으로 평가할 기준 없음
└─ 여러 도구를 사용해야 함 (AI 글쓰기 + SEO 검사)

시간:
├─ 분석/최적화에 시간을 쓸 여유 없음
├─ 한 글에 30분 이상 SEO 최적화 불가능
└─ 월 5개 글도 힘든 상황 → 자동화 원함

비용:
├─ 유료 SEO 도구 (월 10만원+) 불필요
├─ "무료 또는 저가" 원함
└─ 고객 획득 당 ROI 계산 엄격
```

### 2.6 사용 시나리오

#### Scenario A: 신규 글 작성 후 SEO 최적화

```
시간: 월요일 저녁 8시
상황: AI 블로그 기능으로 "기업 소송 변론일 관리법" 글 생성 완료

박준호의 행동:
1. [글 상세 페이지 진입]
   "흠... 이 글이 정말 검색되는 글일까?"
   (SEO 패널이 자동으로 로드됨)

2. [SEO 점수 확인]
   "65/100? 평균 정도네. 빨간 항목들이 눈에 띄네."
   (메타 디스크립션 0/10, 내부 링크 3/10)

3. [빠른 최적화]
   "메타 디스크립션만 빨리 수정하자. 가장 점수 올라갈 것 같은데."
   → 메타 디스크립션 [적용] 클릭
   → 비교 모달에서 제안 검토 (5초)
   → [확인] 클릭 (점수 75/100으로 상승)

4. [만족]
   "좋네! 이 정도면 게시하자."
   → 글 발행 클릭

소요 시간: 3분
만족도: 4.5/5
재사용 의향: 매우 높음 (모든 글마다 사용)
```

#### Scenario B: 기존 글 일괄 최적화

```
시간: 주말 오전 10시
상황: 지난 3개월 작성한 글 15개 중 검색 노출 안 되는 글들이 있음

박준호의 행동:
1. [글 목록 진입]
   → "SEO 점수 50 이하" 필터링
   → 5개 글 선택

2. [첫 번째 글 진입]
   "이 글은 왜 안 될까..."
   (SEO 패널 확인, 점수 52/100)

3. [모든 항목 최적화]
   → [모든 항목 최적화] 클릭
   → 진행률 모달 표시
   → 각 항목마다 제안이 순차적으로 나타남

4. [선택적 수용]
   - 메타 디스크립션: ✓ 수용
   - 키워드 밀도: ✓ 수용 (추가 문장 삽입)
   - 내부 링크: △ 검토 후 수용 (2개 링크 추가)
   - H2 구조: ✓ 수용

5. [완료]
   "점수가 82/100으로 올라갔네! 다음 글도 해보자."
   (남은 4개 글도 같은 방식으로 최적화)

소요 시간: 5개 글 × 10분 = 50분
만족도: 4.0/5 (일부 제안은 직접 수정 필요)
재사용 의향: 높음
```

### 2.7 사용 빈도 및 습관

```
월간 사용 패턴:
├─ 신규 글 작성: 월 3~5회
│  └─ 각 글당 SEO 패널 방문 1회 (필수)
├─ 기존 글 최적화: 월 1~2회
│  └─ 일괄 5~10개 글 (주말 여유 있을 때)
└─ 분석/모니터링: 월 4회 (주 1회, 금요일)

총 월간 방문: 12~15회
활동 시간: 총 2~3시간/월

시간대:
├─ 업무시간: 글쓰기 직후 (14:00~17:00) - 10%
├─ 저녁: 글 검토 (19:00~21:00) - 30%
├─ 주말: 일괄 최적화 (09:00~12:00) - 60%
```

├─ 프리랜서 에디터: 글당 20만원
├─ SEO 분석 도구: 월 15만원
└─ 통합하면 비용이 너무 높음

기술:
├─ 기술팀 부재 (IT 관리자만 있음)
├─ "AI 도구"라고 하면 의심 (정확성 문제)
├─ 보안/법적 책임 우려 높음
└─ 도입 전 검증 필요
```

```
시간: 화요일 오후 3시
상황: 신입 변호사가 "스타트업 투자 계약" 글을 제출했음

이영미의 행동:
1. [마케팅팀과 협력]
   이영미: "이 글을 SEO 최적화된 버전으로 만들어야 하는데,
             새로운 도구가 있어. 점수가 몇 점이나 되나?"

2. [SEO 패널 확인]
   글 링크 열기 → SEO 점수 확인 (48/100)
   "음... 좀 낮네. 최소 70은 넘어야 하는데."

3. [팀원 피드백 생성]
   이영미: "이 글은 다음 부분을 개선해야 해:
           - 메타 디스크립션 추가
           - 내부 링크 추가
           - 키워드 밀도 조정

5. [모니터링]

   - 평균 SEO 점수: 72점 (상승)
   - 글 검토 시간: 40% 감소
   - 블로그 방문: 월 8,000 (증가)

소요 시간: 5분 (초기) → 이후 자동화
만족도: 5/5 (팀 전체 표준화 가능)
재사용 의향: 매우 높음 (정책으로 강제)

### 5.4 Concern (우려)

```
기술:
├─ "AI가 법률 의견을 제시하는 건 아닐까?"
├─ "회사 비밀 정보가 유출되지 않을까?"
└─ "AI 생성 내용의 법적 책임은 누가?"

사용:
├─ 직접 사용 거의 안 함 (팀원이 사용하는 거만 감시)
├─ 도입 전 "안전성 검증" 필수
└─ 약관/면책조항 법무팀 검토 필요
```

## 6. 사용자 여정 맵 (Customer Journey Map)

### 6.1 박준호 (개인 변호사) - Persona 중심

```
Stage 1: 인식 (Awareness)
├─ 채널: 친구 변호사 추천, 블로그 커뮤니티
├─ 메시지: "SEO 점수를 자동으로 올려주는 AI 도구"
├─ 감정: 관심 (호기심)
└─ CTA: "무료로 해보기"

Stage 2: 고려 (Consideration)
├─ 행동: 첫 글로 직접 테스트
├─ 메시지: "45초 안에 SEO 점수 분석"
├─ 감정: 기대감 (효과가 있을까?)
└─ Friction: 로그인 과정, 약관 동의

Stage 3: 전환 (Conversion)
├─ 행동: 글 게시 시 SEO 점수 > 60점 달성
├─ 메시지: "축하합니다! 네이버 검색 최적화 완료"
├─ 감정: 만족감
└─ CTA: "다음 글도 최적화하기"

Stage 4: 사용 (Usage)
├─ 행동: 모든 신규 글에 SEO 최적화 적용
├─ 메시지: "월 평균 블로그 방문 500 → 2,000"
├─ 감정: 확신 (이게 효과 있구나!)
└─ Retention: 높음 (매달 사용)

Stage 5: 충성 (Loyalty)
├─ 행동: 친구 변호사들에게 추천
├─ 메시지: "이 도구 덕분에 신규 고객 3명 확보했어"
├─ 감정: 감사, 자부심
└─ Advocacy: 입소문 확산

각 Stage별 주요 접점 (Touchpoint):
├─ Website → 트라이얼 → 이메일 → 대시보드 → 모니터링
└─ 지속적 피드백 (주 1회 성과 이메일)
```

## 8. 페르소나별 메시징 전략

### 8.1 박준호에게 말하는 방식

```
"박준호님, 계속 하던 대로 글을 쓰세요.
우리가 SEO 최적화 부분은 맡겨드릴게요.
3분만에 점수 올려주니까요."

강점:
├─ 간단함 (복잡한 설명 X)
├─ 빠름 (시간 절약)
├─ 효과 (결과 중심)
└─ 증명 (수치 제시)
```

### 8.2 이영미에게 말하는 방식

```
"이영미님, 이 도구로 팀 전체의 글 품질을 표준화할 수 있어요.
객관적인 점수로 피드백을 주고,
마케팅팀의 검토 시간도 40% 줄일 수 있습니다."

강점:
├─ 표준화 (일관성)
├─ 효율성 (시간 절감)
├─ 데이터 (성과 추적)
└─ 관리 (팀 통제)
```

### 8.3 김수진에게 말하는 방식

```
"김수진님, 이제 변호사들이 SEO를 알아야 한다는 부담에서 벗어나세요.
도구가 객관적으로 점수를 매겨주니까요.
당신은 데이터 분석만 하면 됩니다."

강점:
├─ 자동화 (수동 작업 감소)
├─ 데이터 (의사결정 근거)
├─ 신뢰 (객관성)
└─ 효율 (시간)
```

## 9. 페르소나별 Onboarding 계획

### 9.1 박준호 (개인 변호사) - 간단함

```
1단계: 가입 (1분)
└─ "로그인하고 첫 글 열기"

2단계: 분석 (자동)
└─ SEO 패널 자동 로드

3단계: 성공 경험 (3분)
└─ 메타 디스크립션 [적용] 클릭
└─ "점수 10점 올랐어요! 🎉"

4단계: 고급 기능 (선택)
└─ "모든 항목 최적화" 다음에 소개
```

### 9.2 이영미 (로펌 변호사) - 관리 중심

```
1단계: 로펌 설정 (5분)
└─ 팀 초대, 권한 설정

2단계: 표준 설정 (10분)
└─ "최소 SEO 점수 기준: 70"
└─ "글 발행 전 필수 확인"

3단계: 팀 교육 (30분)
└─ 변호사들에게 가이드 공유
└─ 예시 글로 데모

4단계: 모니터링 (지속)
└─ 월간 리포트 자동 생성
└─ 대시보드에서 팀 성과 추적
```

### 9.3 김수진 (마케팅 담당자) - 분석 중심

```
1단계: 연동 (5분)
└─ Google Analytics 연동 (선택)
└─ 데이터 수집 시작

2단계: 대시보드 (10분)
└─ 맞춤 보고서 설정
└─ KPI 정의 (점수 vs 트래픽)

3단계: 분석 (지속)
└─ 주간 리포트 받기
└─ "높은 점수 글" = "더 많은 방문" 증명

4단계: 최적화 (지속)
└─ 최적화 방향 제시
└─ 변호사들에게 피드백
```
